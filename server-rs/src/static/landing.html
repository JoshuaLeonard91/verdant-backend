A familiar shape, cut down to the parts that matter.

Verdant is not a public feed or algorithmic social app. It is organized chat: servers, text channels, voice channels, member lists, role colors, uploads, and notification controls.

Community

Start with one server and grow carefully.

Create channels, assign roles, manage invites, and keep private spaces visible only to people who should be there.

Desktop

A native app first, not a browser tab you lose.

The Windows desktop client is the first public target. The website stays focused on discovery, status, documentation, and support.

Automation

Bots are self-hosted and permissioned.

Use bot tokens, scoped APIs, rich feed cards, and channel cards without handing Verdant your automation secrets.

Feeds for announcements, channels for discussion.

Announcements can be composed as structured cards with headings, text, tables, images, dividers, and buttons. A release bot can post the formatted note, then link people into a text channel for conversation.

Release notes
Post to a feed with a rich card and a channel button for discussion.
Moderation helpers
Connect over the bot gateway when the bot needs to receive events.
Status cards
Send clean, formatted channel cards from a self-hosted service.
External hooks
Validate incoming webhooks in your own bot, then post through Verdant's scoped API.

Soft launch posture: account creation is open, but email verification is required before the app unlocks.

Privacy posture: the app stores what it needs for chat features, file delivery, security, and abuse prevention.

Platform posture: Windows is the first public desktop target. Other desktop and mobile paths can come later without making the website pretend they exist today.

Soft launch access

Existing desktop installs receive production updates through the built-in updater. New access is handled through the soft launch channel.
